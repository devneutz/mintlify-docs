---
title: "Creating an Event"
description: "Set up an event, add shifts, and publish it for volunteers"
---

An event is anything your club needs helpers for — a tournament, a club
festival, a work day. Creating one takes two parts: the event itself (title,
description, reminder and data-retention settings), and the shifts inside it
(the actual roles volunteers sign up for).

<Note>
  New events start as a **draft**. Volunteers can't see or sign up for a
  draft event — you publish it explicitly once it's ready.
</Note>

<Steps>
  <Step title="Open the Events page">
    Sign in to your club's admin area and go to **Events** in the top nav.
    Click **New event**.
  </Step>
  <Step title="Fill in the event details">
    | Field | Notes |
    | --- | --- |
    | Title | Required, shown to volunteers on the public page. |
    | Description | Optional — context for volunteers (what the event is, what to bring). |
    | Location | Optional — free text, e.g. the venue name. |
    | Reminder email (days before shift start) | How long before their shift each volunteer gets a reminder email. Default 1 day. |
    | Delete volunteer data (days after the event ends) | The retention window: how long signup names/emails are kept after the event's last shift before they're automatically anonymized. Default 30 days. |

    Click **Create event**.
  </Step>
  <Step title="Add shifts">
    On the event page, open **+ Add shift** and fill in:

    | Field | Notes |
    | --- | --- |
    | Task | The role, e.g. "Grill" or "Ticket booth". Free text. |
    | Slots | How many volunteers this shift needs. |
    | Start / End | The shift's time window. |
    | Location | Optional — useful if it differs from the event location. |
    | Notes | Optional — anything a volunteer should know before signing up. |

    Click **Add**, then repeat for every shift the event needs. You can add
    more shifts later at any time, published or not.
  </Step>
  <Step title="Publish the event">
    Once you have at least one shift, click **Publish** at the top of the
    event page. This makes the event's public page live at its share link.

    Use **Copy link** (or **Public link ↗** to preview it) to get the URL to
    share with your volunteers.
  </Step>
</Steps>

<Tip>
  If your club has [teams](/en/guides/managing-teams) set up
  (e.g. "Junioren", "Damen 1"), each team gets its own share link with the
  team preselected — handy for forwarding to a specific group's group chat.
  Find these under **Team links** on the event page.
</Tip>

## What happens after you publish

- The event page becomes visible to anyone with the link.
- Volunteers sign up with just their name and email — no account required —
  and get a confirmation email with a link to manage or cancel their signup.
- You can keep editing the event and its shifts, and you can revert to draft
  at any time with **Revert to draft** if you need to pull it down.
